Issue with MIUI Icon Cache

Apr 18th 2015

Issue

It's easy to change the icon of an Android App. All you need to do is simply replace the ic_launcher.png file with your new design. But in MIUI(An 3rd-party Android ROM from XiaoMi co. which might be the best ROM there is by now. Really popular in China. Also popular in India, America and many other countries.), things get a little bit tricky. When you install the app, you won't see any changes of the icon. it's still using the default Android icon instead of you new design. All codes are right. You try to install again or change the file. But nothing works.

Effects

As far as we know, this issue was brought by MIUI 6. Previous versions should be working fine. As Consequence, when your app has major updates, your users will be running the latest program but seeing the old icon, until they find a way to fix it. Meanwhile it also brought trouble for Android developers cause they got one more problem to deal with.

Cause

The reason of this is that MIUI has a cache for icons. images in this cache won't disappear even after you uninstalled the app or ran a system clean up program. That is why you can't solve this problem by reinstall or change the image.

Sulution

To solve this problem is easy. Just go into the "theme" app, choose another theme and apply, then switch back to the previous theme. Now you should be seeing the new icon. Mission accomplished. Changing theme will refresh the icon cache(Someone solve this by rebooting the phone, you can try your onw.)

Folk Prescription

There're some folk prescription claim to be the solution but actually doesn't work. Here I'll post some of them that I know for your information.

Folk prescription 1: If you already had root authority, go to "/data/system/customized_icons", find your icon and delete it. Theoretically this should be working, but seems MIUI's icon cache is not here. We tried but not working.

Folk prescription 2: Change the package name of the project(or module if you are using Android Studio) and reinstall. A new package name means a new App. Old cache has no effect on new apps. Sounds true but still not working.
