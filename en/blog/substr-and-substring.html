Differences between substr() and substring()

Mar 15th 2015

There're these two frequently used but confusing functions in JavaScript: substr() and substring(). Both are method of String that return a subset of a string.

For detail references of these two functions, see: String.prototype.substr() and String.prototype.substring().

Both function accept two arguements to define the begining and end of the subset. The first arguement works the same ( with a little difference in action ): the start point. Characters starting from the start point will be included as the subset of the string. But the second arguement differs a lot. substr() uses length to define how many characters should be included, while substring() directly defines the end point. Any character starting from the end point will not be included.

substr ( start [ , length ] )

  • start
    1. Location at which to begin extracting characters.
    2. If ≥ the length of the string, an empty string will be returned.
    3. If < 0, then count from the right.
    4. If < 0 but the absolute value is ≥ the length of the string, it will be treated as 0.
  • length (optional)
    1. The number of characters to extract.
    2. If the extracting process reaches the end of the string with fewer characters. it will return as much as there is.
    3. If ≤ 0, an empty string will be returned.
    4. If omitted, it extracts characters to the end of the string.

For Example:

var str = "abcgefghij";

console.log('(1, 2): '   + str.substr(1, 2));   // '(1, 2): bc'
console.log('(-3, 2): '  + str.substr(-3, 2));  // '(-3, 2): hi'
console.log('(-3): '     + str.substr(-3));     // '(-3): hij'
console.log('(1): '      + str.substr(1));      // '(1): bcdefghij'
console.log('(-20, 2): ' + str.substr(-20, 2)); // '(-20, 2): ab'
console.log('(20, 2): '  + str.substr(20, 2));  // '(20, 2): '

substring ( start [ , end ] )

  • start
    1. An integer between 0 and the length of the string, specifying the offset into the string of the first character to include in the returned substring.
    2. If ≥ the length of the string. an empty string will be returned.
  • end (optional)
    1. An integer between 0 and the length of the string, which specifies the offset into the string of the first character not to include in the returned substring.
    2. If > the length of string, it is treated as if it were the length.
    3. If omitted, it extracts characters to the end of the string.
  • 特殊情况
    1. If either arguement is < 0 or is NaN, it is treated as 0.
    2. If start == end, an empty string will be returned.
    3. If start > end, exchange thier value.

For Example:

var str = 'abcdefg';

// Displays 'abc'
console.log(str.substring(0, 3));
console.log(str.substring(3, 0));

// Displays 'efg'
console.log(str.substring(4, 7));
console.log(str.substring(7, 4));

// Displays 'abcdef'
console.log(str.substring(0, 6));

// Displays 'abcdefg'
console.log(str.substring(0, 7));
console.log(str.substring(-2, 7));
console.log(str.substring(0, 10));

Immutable String

Like in many other programming languages, String is immutable in JavaScript too.substr() and substring() only return the subset of the string. You gonna need another variable if you wanna store the result. Else it'll be just an oputput.

Boundary crossing friend: slice ( start [ , end ] )

Except for the two main character of this article, there's another function in JavaScript that can be used to get a subset of a string. It's their friend slice(). The reason why I called "Boundary Crpssing" is that it's defined in both String and Array.

slice() somehow works like the combination of substr() and substring(). It uses starting point and end point to define what to extract, just like substring(). But if either arguement is negative, it's not treated as 0, instead it starts the counting from the right. This works the same as substr() (Nothice that it works for both arguements in slice(), while only the first arguement in substr() works like this). The second arguement is also optional. If omitted or is greater than the length of the rest string. The subset ends with the last character of the string. If the first arguement is greater than the second one, they wont exchange. An empty string will be returned.

slice() works the same in Array. Only the it's array elements. Anyway, String is somehow another kind of array in JavaScript. So it's not strange to see they share some functions.

Why three

substr(), substring() and slice() provided us 3 different ways to extract a string: By length, form the positive direction, from both end. We can choose whatever works the best for our project.
